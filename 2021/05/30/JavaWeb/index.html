HTML书写样例

HTML标签介绍

  • 标签的格式<标签名>封装的数据</标签名>
  • 标签名对大小写不敏感
  • 标签有各自的属性
    • 基本属性：bgcolor，修改简单的样式效果
    • 事件属性：onclick=”alert(‘KK好帅！’)”，可以设置事件响应后的代码
  • 标签有单标签和双标签
    • 单标签：<标签名 />
    • 双标签 <标签名> … </标签名>

标签的语法

  • 标签不能交叉嵌套
  • 标签必须正确关闭(闭合)
  • 属性必须有值，属性值必须加引号
  • 注释不能嵌套

iframe标签

iframe标签可以在一个html页面上打开一个小窗口去加载一个单独的页面

表单标签

表单就是html页面中，用来收集用户信息的所有元素集合，然后把这些信息发送给服务器

表单格式化对齐

表单提交

提交时没有发送给服务器的情况

  • 表单项没有name属性
  • 单选、复选都需要添加value属性
  • 表单项不在提交的form标签中

GET和POST请求

  • GET
    • 浏览器的地址值是action属性[+？+请求参数]
    • 不安全
    • 有数据长度的限制
  • POST
    • 浏览器的地址值只有action属性
    • 相对于GET请求要安全
    • 理论上没有数据长度的限制

Other

  • div标签：默认独占一行
  • span标签：长度是封装数据的长度
  • p标签：默认会在段落的上方和下方各空出一行，如果有就不再空

CSS

层叠样式表单，用于（增强）控制网页样式并允许将样式信息和网页内容分离的一种标记性语言

  • p：选择器，根据选择器决定受css样式影响的HTML元素（标签）
  • font-size：每个属性都会有一个值，这里是80px

CSS和HTML结合

方法1

在标签的style属性上设置”key:value value;”，修改标签样式

缺点

  • 样式和标签多了代码量非常庞大
  • 可读性非常差
  • css代码复用性差

方法2

在head标签中，使用style标签来定义css的样式

缺点

  • 只能在同一页面内复用代码
  • 维护起来不方便

方法3

把css样式写成一个单独的css文件，再通过link标签在head里引入即可复用

JavaScript

主要是用于页面的数据验证

特点

  • 交互性：实现信息的动态交互
  • 安全性：不允许直接访问本体硬盘
  • 跨平台性：可以解释JS的浏览器都可以执行，和平台无关

JavaScript和html代码结合

方式1

只要在head标签中，或者在body标签中，使用script标签写JavaScript代码

方式2

使用script引入外部的js文件来执行
